# 蛋糕系统 - 管理员后台管理功能需求文档

## 1. 需求概述

### 1.1 项目背景
本系统是一个蛋糕销售电商平台，面向普通用户提供蛋糕商品浏览、购买等服务。为了支撑平台运营，需要开发一套管理员后台管理系统，用于管理用户、商品、订单等核心业务数据。

### 1.2 系统定位
管理员后台是面向平台运营人员的内部管理系统，主要用于：
- 用户信息管理与审核
- 商品信息维护
- 订单处理与发货
- 数据统计与分析

### 1.3 目标用户
系统管理员、运营人员

## 3. 数据模型

### 3.1 用户表（user）

| 字段名 | 类型 | 含义 | 约束 |
| :--- | :--- | :--- | :--- |
| id | int | 用户ID | 主键，自增 |
| username | varchar | 用户名 | 非空，唯一 |
| email | varchar | 邮箱 | 非空 |
| password | varchar | 密码 | 非空 |
| name | varchar | 真实姓名 | - |
| phone | varchar | 联系电话 | - |
| address | varchar | 收货地址 | - |
| isadmin | varchar | 是否管理员 | 0/1 |
| isvalidate | varchar | 是否已审核 | 0/1 |

### 3.2 商品表（goods）

| 字段名 | 类型 | 含义 | 约束 |
| :--- | :--- | :--- | :--- |
| id | int | 商品ID | 主键，自增 |
| name | varchar | 商品名称 | 非空 |
| cover | varchar | 封面图片URL | - |
| image1 | varchar | 详情图片1 | - |
| image2 | varchar | 详情图片2 | - |
| price | varchar | 单价 | 非空 |
| intro | varchar | 商品介绍 | - |
| stock | int | 库存数量 | 默认100 |
| type_id | int | 分类ID | 外键关联 type 表 |

### 3.3 订单表（order）

| 字段名 | 类型 | 含义 | 约束 |
| :--- | :--- | :--- | :--- |
| id | varchar | 订单编号 | 主键 |
| total | float | 订单总金额 | 非空 |
| amount | int | 商品总数量 | 非空 |
| status | int | 订单状态 | 2=已付款,3=已发货,4=已完成 |
| paytype | int | 支付类型 | 1=微信,2=支付宝 |
| name | varchar | 收货人姓名 | 非空 |
| phone | varchar | 收货人电话 | 非空 |
| address | varchar | 收货地址 | 非空 |
| datetime | varchar | 创建时间 | 非空 |
| user_id | int | 用户ID | 外键关联 user 表 |

### 3.4 订单明细表（order_item）

| 字段名 | 类型 | 含义 | 约束 |
| :--- | :--- | :--- | :--- |
| id | int | 明细ID | 主键，自增 |
| order_id | varchar | 订单编号 | 外键关联 order 表 |
| good_id | int | 商品ID | 外键关联 goods 表 |
| name | varchar | 商品名称 | 非空 |
| price | float | 单价 | 非空 |
| amount | int | 数量 | 非空 |

### 3.5 分类表（type）

| 字段名 | 类型 | 含义 | 约束 |
| :--- | :--- | :--- | :--- |
| id | int | 分类ID | 主键，自增 |
| name | varchar | 分类名称 | 非空，唯一 |

| :--- | :--- | :--- |
| /user/login | POST | 管理员登录 |

**请求参数：**

| 参数名 | 类型 | 必填 | 含义 |
| :--- | :--- | :--- | :--- |
| username | String | 是 | 用户名 |

## 8. 业务流程图

### 8.1 管理员登录流程

```
用户访问登录页 → 输入用户名密码 → 验证身份 → 判断isadmin → 跳转对应页面
                                           ↓
                                    isadmin=1 → 管理员控制台
                                    isadmin=0 → 用户首页
```

### 8.2 订单处理流程

```
订单创建 → 等待付款 → 已付款(2) → 发货(3) → 已完成(4)
                          ↓
                    管理员点击发货
```
