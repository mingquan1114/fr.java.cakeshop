
# 骑士端需求分析文档

## 1. 项目概述

### 1.1 背景
基于现有蛋糕商城项目，新增骑士端模块，实现订单配送全流程管理。骑士端作为连接商家与用户的重要环节，负责订单的取货、配送和送达确认。

### 1.2 项目目标
- 提供骑士登录和身份验证功能
- 实现订单状态流转管理（待接单→待取货→配送中→已完成）
- 提供订单详情查看和配送操作功能
- 实现骑士收入统计和消息通知功能

### 1.3 技术架构
| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Spring Boot + Thymeleaf |
| 后端框架 | Spring Boot 2.2.6 |
| 数据库 | MySQL 8.0+ |
| ORM框架 | MyBatis Plus 3.5.0 |
| 构建工具 | Maven |

## 2. 功能需求分析

### 2.1 核心功能模块

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| 骑士登录 | 骑士账号密码登录、退出 | 高 |
| 订单管理 | 待接单、待取货、配送中、已完成订单列表 | 高 |

## 3. 数据流与状态流转

### 3.1 订单状态流转图

```mermaid
stateDiagram-v2
    [*] --> 待付款: 用户下单
    待付款 --> 待接单: 用户支付成功(status=2)
    待接单 --> 待取货: 骑士确认接单
    待取货 --> 配送中: 骑士确认取货
    配送中 --> 已完成: 骑士确认送达(status=4)
```

### 3.2 订单状态映射表

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待付款 | 用户未支付 |
| 2 | 待接单 | 用户已支付，等待骑士接单 |
| 3 | 待取货 | 骑士已接单，等待取货 |
| 4 | 配送中 | 骑士已取货，正在配送 |
| 5 | 已完成 | 订单已送达 |

## 8. 与现有系统的集成

### 8.1 订单状态联动

| 操作方 | 操作 | 订单状态变化 | 骑士端影响 |
|--------|------|--------------|------------|
| 用户 | 支付成功 | 0 → 2 | 待接单列表新增订单 |
| 骑士 | 确认接单 | 2 → 3 | 待接单→待取货 |
| 骑士 | 确认取货 | 3 → 4 | 待取货→配送中 |
| 骑士 | 确认送达 | 4 → 5 | 配送中→已完成 |

### 8.2 数据交互流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Server as 服务器
    participant Knight as 骑士端

    User->>Server: 支付订单
    Server->>Server: 更新订单状态为待接单(2)
    Server->>Knight: 推送新订单通知
    Knight->>Server: 查询待接单订单列表
    Server-->>Knight: 返回订单列表
    Knight->>Server: 确认接单
    Server->>Server: 更新订单状态为待取货(3)
    Server-->>Knight: 接单成功
    Knight->>Server: 确认取货
    Server->>Server: 更新订单状态为配送中(4)
    Server-->>Knight: 取货成功
    Knight->>Server: 确认送达
    Server->>Server: 更新订单状态为已完成(5)
    Server-->>Knight: 送达成功
```
